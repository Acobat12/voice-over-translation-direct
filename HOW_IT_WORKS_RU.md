# Как это работает

Этот проект не делает распознавание речи, перевод и синтез локально в браузере.
Он работает как клиент к готовому VOT/Yandex-стеку: находит видео на странице, получает для него пригодный URL, отправляет метаданные на сервер перевода и затем подмешивает полученную озвучку поверх исходного плеера.

## Коротко по шагам

1. Скрипт запускается на странице.
2. `src/index.ts` поднимает bootstrap и включает `VideoObserver`.
3. `src/utils/VideoObserver.ts` ищет `video` в обычном DOM, в Shadow DOM и в динамически добавленных узлах.
4. Для найденного видео выбирается конфигурация сайта:
   - сначала стандартные сайты из `@vot.js/ext`
   - потом кастомные сайты из `src/config/extraSites.ts`
   - если сайт неизвестен, но на странице есть `video`, включается generic-режим с `host: "custom"`
5. `src/core/videoManager.ts` собирает `videoData`:
   - `url`
   - `videoId`
   - `title`
   - `duration`
   - язык оригинала
   - субтитры, если они доступны
6. Для нестандартных сайтов берётся лучший источник в таком порядке:
   - перехваченный manifest (`m3u8`/`mpd`) через `manifestSniffer`
   - `video.currentSrc`
   - `video.src`
   - уже найденный URL от `@vot.js/ext`
   - URL текущей страницы как fallback
7. `src/core/translationHandler.ts` нормализует URL и вызывает `votClient.translateVideo(...)`.
8. После ответа сервера подключается переведённая аудиодорожка, а оригинальная при необходимости приглушается.

## Какие файлы важны

- `src/index.ts`
  Точка входа. Bootstrap, выбор сайта, создание `VideoHandler`.

- `src/bootstrap/videoObserverBinding.ts`
  Склеивает `VideoObserver` и `VideoHandler`.

- `src/utils/VideoObserver.ts`
  Следит за появлением/исчезновением видео и фильтрует мусорные/рекламные плееры.

- `src/config/extraSites.ts`
  Быстрый способ добавить ещё один домен, если на нём есть обычный HTML5-видеоплеер.

- `src/config/playerSelectors.ts`
  Набор generic-селекторов контейнера плеера.

- `src/core/videoManager.ts`
  Собирает `videoData`, определяет язык, заголовок, субтитры и URL для перевода.

- `src/core/translationHandler.ts`
  Отправляет запрос на перевод и управляет жизненным циклом аудиодорожки перевода.

- `src/utils/manifestSniffer.ts`
  Ловит `m3u8`/`mpd`, когда плеер не отдаёт прямой `src`.

- `src/core/customSiteResolvers.ts`
  Тут делаются специальные хаки для сайтов, где URL видео надо выковыривать отдельно.

## Что значит "работает на любом сайте"

Практически это означает:

- страница должна быть доступна userscript/extension
- на странице должен быть настоящий `HTMLVideoElement`
- у видео должен быть доступен прямой `src`, `m3u8`, `mpd` или другой URL, который можно передать на сервер перевода

Если этих условий нет, generic-режим не поможет.

## Когда generic-режим сработает без дописывания кода

Ничего добавлять не нужно, если:

- на странице есть обычный `<video>`
- `video.currentSrc` уже содержит рабочий media URL
- или manifest можно поймать через сеть
- контейнер плеера находится по одному из селекторов из `src/config/playerSelectors.ts`

После изменений в `src/headers.json` скрипт теперь запускается на любых обычных `http/https` страницах, поэтому для многих сайтов отдельная запись в `match` больше не нужна.

## Когда нужно дописывать код

Нужно вмешательство, если:

- видео лежит внутри iframe-цепочки и нужный URL есть только во внутреннем фрейме
- сайт использует нестандартный контейнер и overlay не к чему привязать
- `video.src` это `blob:` и manifest не удаётся перехватить
- сайт требует особый способ получить title, subtitle или canonical URL

Обычно порядок такой:

1. Проверить, появился ли `host: "custom"` и нашёлся ли `video`.
2. Проверить лог `"[VOT][custom:getVideoData]"`.
3. Если URL плохой, добавить логику в `src/core/customSiteResolvers.ts`.
4. Если overlay не прикрепляется к плееру, расширить `src/config/playerSelectors.ts`.
5. Если сайт надо матчить или приоритизировать явно, добавить запись в `src/config/extraSites.ts`.

## Почему некоторые сайты всё равно не заработают

- DRM/EME-плееры, где нет доступного media URL
- экзотические canvas/WebRTC-плееры без обычного `video`
- сайты, где аудио/видео режется токенами так, что сервер перевода не может скачать поток
- ролики длиннее лимита сервиса перевода

То есть "любой сайт" в этом проекте означает не магию, а максимально широкий generic fallback для HTML5-видео.

## Самая важная мысль

Если у сайта есть нормальный `video` и доступный media URL, то новый код часто вообще не нужен.
Если URL спрятан, то задача уже не "сделать переводчик", а "вытащить корректный URL видео и отдать его в существующий переводчик".
